Konsep Kunci: Bagaimana traceId Tetap Sama?

Setiap service membaca header traceparent dari request/message yang masuk, lalu membuat child transaction yang mewarisi traceId. Jika header tidak ada, dibuat trace baru.

Mekanisme per Layer

🌐

Browser → HTTP

RUM agent menambahkan header traceparent secara otomatis ke semua Axios request berdasarkan konfigurasi distributedTracingOrigins. Tidak perlu kode manual.

⚙️

ASP.NET Core → HTTP

AddAllElasticApm() otomatis membaca header traceparent dari setiap HTTP request dan membuat child transaction. Seluruh proses transparan tanpa kode tambahan.

📤

Publish ke RabbitMQ

Header elastic-apm-traceparent ditulis manual ke AMQP message properties. Nilainya diambil dari span.OutgoingDistributedTracingData?.SerializeToString().

📥

Consume dari RabbitMQ

Header dibaca manual, di-parse dengan DistributedTracingData.TryDeserializeFromString(), lalu dipakai sebagai parameter ke Agent.Tracer.StartTransaction(..., tracingData).

🔑

Format traceparent (W3C)

00-7152c87c...4434a-a1b2c3d4-01
version · traceId (16 bytes) · parentSpanId (8 bytes) · flags
traceId selalu sama; hanya parentSpanId yang berubah per hop.

🆚

Trace baru vs. child transaction

TryDeserializeFromString() memvalidasi string. Jika valid → child transaction (traceId diwarisi). Jika null atau corrupt → trace baru (traceId random). Tidak ada exception.
